# Contributing Guidelines

Thank you for your interest in contributing to this project! This document outlines the process and standards for contributing to the **Information Technology Project Management** course repository.

## 📌 How to Contribute

### 1. Fork and Clone

1. Fork this repository to your own GitHub account
2. Clone your fork locally:
   ```bash
   git clone https://github.com/<your-username>/main.git
   cd main
   ```

### 2. Create a Branch

Always work on a new branch. Use a descriptive name:

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/issue-description
```

### 3. Make Your Changes

- Follow the existing documentation style and structure
- Keep commits small and focused
- Write clear, descriptive commit messages

### 4. Commit Your Changes

Use meaningful commit messages that describe what and why:

```bash
git add .
git commit -m "Add project milestone tracker to docs"
```

### 5. Push and Open a Pull Request

```bash
git push origin feature/your-feature-name
```

Then open a Pull Request on GitHub with:
- A clear title
- A description of what was changed and why
- References to any related issues

## ✅ Contribution Standards

- **Documentation**: Ensure all documentation is clear, accurate, and up-to-date
- **Commit Messages**: Use present tense and be descriptive (e.g., "Add risk management section")
- **Branches**: Never commit directly to `main`; always use a feature branch
- **Pull Requests**: All changes must be reviewed before merging

## 🐛 Reporting Issues

If you find a problem or have a suggestion:

1. Check if the issue already exists in the [Issues](../../issues) tab
2. If not, open a new issue with:
   - A descriptive title
   - A clear description of the problem or suggestion
   - Steps to reproduce (if applicable)

## 📋 Code of Conduct

- Be respectful and inclusive in all communications
- Provide constructive feedback
- Collaborate openly and transparently

## 🙋 Questions?

If you have questions, open a [GitHub Discussion](../../discussions) or reach out to the project team directly.
