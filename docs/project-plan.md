# Project Plan

## Project Title
Information Technology Project Management – Course Project

## Project Overview

This project is undertaken to fulfill the graduation requirements for the course **"Information Technology Project Management"**. It applies core IT project management frameworks and practices through a hands-on project documented and managed via GitHub.

## Scope Statement

### In Scope
- Development and maintenance of project documentation
- Application of project management methodologies (initiation, planning, execution, monitoring, closure)
- Version control and collaboration using GitHub
- Delivery of all required project artifacts

### Out of Scope
- External stakeholder deployments
- Production infrastructure setup

## Schedule

| Phase | Task | Start Date | End Date | Owner |
|-------|------|------------|----------|-------|
| Initiation | Define scope and objectives | Week 1 | Week 1 | Project Manager |
| Initiation | Identify stakeholders | Week 1 | Week 1 | Project Manager |
| Planning | Develop project plan | Week 2 | Week 2 | Project Manager |
| Planning | Set up GitHub repository | Week 2 | Week 2 | Team Lead |
| Planning | Define milestones and deliverables | Week 2 | Week 2 | Project Manager |
| Execution | Create project documentation | Week 3 | Week 5 | All |
| Execution | Implement deliverables | Week 3 | Week 6 | Developer(s) |
| Monitoring | Track progress and manage risks | Week 3 | Week 7 | Project Manager |
| Closure | Final review and submission | Week 7 | Week 8 | All |

## Resource Plan

| Resource | Role | Allocation |
|----------|------|------------|
| Project Manager | Planning, coordination, reporting | 100% |
| Team Lead | Technical oversight, code reviews | 100% |
| Developer(s) | Implementation, documentation | 100% |
| QA/Tester | Testing, quality assurance | 50% |

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Schedule delays | Medium | High | Regular check-ins; adjust scope if needed |
| Unclear requirements | Low | High | Early stakeholder engagement; document all requirements |
| Team member unavailability | Low | Medium | Cross-train team; document processes |
| Technical challenges | Medium | Medium | Research solutions early; seek instructor guidance |

## Communication Plan

| Audience | Communication Type | Frequency | Channel |
|----------|--------------------|-----------|---------|
| Team | Status updates | Weekly | GitHub Issues / Discussions |
| Instructor | Progress reports | Bi-weekly | Course platform |
| Stakeholders | Project updates | As needed | Email / Presentation |

## Quality Management

- All documentation reviewed by at least one team member before merging
- GitHub Pull Requests used for all changes
- Deliverables checked against course requirements before submission
